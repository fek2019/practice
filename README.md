# Watch Lab Frontend (Next.js)

Полный fullstack-проект сервиса мастерской по ремонту часов:
- главная страница;
- каталог услуг с фильтрами;
- страница мастеров + детальная карточка;
- многошаговая запись онлайн (5 шагов);
- кабинеты: клиент, мастер, админ;
- страница контактов;
- адаптивная верстка (mobile first).

## Стек

- **Frontend:** Next.js (App Router) + React + TypeScript
- **Backend:** Next.js API routes
- **БД:** локальная SQLite (`better-sqlite3`) по умолчанию; опционально — Supabase или in-memory mock
- **Авторизация:** подписанные session-токены, scrypt-хеши паролей

## Запуск

1. Установить Node.js 20+.
2. Установить зависимости:
   ```bash
   npm install
   ```
3. Запустить проект:
   ```bash
   npm run dev
   ```
4. Открыть: `http://localhost:3000`.

При первом запуске автоматически создаётся `data/watchlab.db`, разворачивается схема и заливаются демо-данные. Никакой ручной миграции не требуется.

## Источники данных

Backend поддерживает три режима, переключаются через `APP_DATA_SOURCE`:

| Значение | Описание |
| --- | --- |
| `sqlite` *(default)* | Реальная локальная БД в файле, без внешних сервисов |
| `mock` | Все данные в памяти, теряются при рестарте |
| `supabase` | Внешний Supabase-проект |

Подробнее — в `BACKEND_CHANGES.md` и `.env.example`.

## Где хранится БД

- Файл: `data/watchlab.db` (плюс WAL-файлы рядом).
- Перенос: скопируйте каталог `data/` на новую машину — это и есть вся БД.
- Сброс: остановите проект, удалите `data/`, перезапустите — схема и сиды пересоздадутся.
- Каталог `data/` в `.gitignore`, в репозиторий не попадает.
